## Why

The platform's VPS paper-intake now runs the 043 trust-model verifier: every `submitPaperCandidate` is checked for a signed `backtest-evidence/v1` artifact (Ed25519, keyId allowlist). Fail-closed matrix: no resolvable artifact → `quarantined`; bad signature → `rejected evidence_signature_invalid`; `verdict !== 'passed'` → `rejected backtest_not_passed`; `bundleHash` / `{datasetRef,window,symbols,timeframe}` mismatch → `rejected` (hash-pin / scope). Lab's real WFO/strategy_baseline champion runs — the ones `paperStartHandler` promotes to paper — carry **no** signed evidence today, so every real submission against the enforced VPS intake would be quarantined. Lab cannot close this gap alone: the whole point of the signature is that a **trusted backtester** attests to the verdict; lab signing its own evidence would be "judge and defendant," which the trust model explicitly forbids (see the 079 design doc §1).

## Current gap — corrected against what's actually in this repo today (file:line)

The lab-side 079 design doc's grounding research (written before this handoff) characterized the gap as "`signEvidence` is CLI-only with no run-pipeline callers." **That's no longer fully accurate** — while writing this handoff we traced the actual pipeline and found real-run signing already exists, just unreachable from lab's normal submission path. The corrected picture:

1. **A real-run signing pipeline already exists, but it's gated on a code path lab never uses.** `apps/backtester/src/jobs/worker.ts:557-590` — the "E4 evidence block" — calls `produceStrategyEvidence` (`apps/backtester/src/evidence/produce-strategy-evidence.ts:69`) with the **real** candidate run's outcome and **real** submitted bundle bytes, and sets `evidenceRef` on the wire result (`worker.ts:589`). But the block only runs `if (claimed.request.curatedBaselineRef !== undefined && deps.evidenceSigningKey !== undefined)` (`worker.ts:562`) — it requires a **second, in-process "curated baseline" twin run** of the same period to prove sandbox/trusted equivalence before signing (`produce-strategy-evidence.ts` runs a full gate → twin-equivalence → verdict chain). Lab's actual strategy-lane submission (`trading-lab/src/research/backtester-strategy-experiment-run-executor.ts`, via `ResearchPlatformPort.submitStrategyResearchRun`) **never sets `curatedBaselineRef`** — zero occurrences of that field anywhere in trading-lab. So for every real WFO/strategy_baseline holdout run lab submits today, this block silently never fires and `evidenceRef` stays `undefined`. The gap for lab isn't "no signing pipeline" — it's "the signing pipeline demands a twin-equivalence proof lab's submissions don't (and shouldn't have to) provide."
2. **Delivery is already half-solved — `evidenceRef` exists and is fetchable, it's just never populated for lab.** `RunResultSummary` (`packages/sdk/src/contracts/run.ts:123`) already carries `readonly evidenceRef?: ArtifactReference` (`run.ts:132`) as a distinct, additive field from `readonly evidence: RunEvidence` (`run.ts:128`, type at `run.ts:88` — confirmed `{seed, contractVersion, moduleVersions, datasetRef, datasetFingerprint?, bundleHash?}`, no signature, matching the original grounding claim exactly for *that* field). `evidenceRef` resolves to a real artifact via the existing `GET /v1/runs/:runId/artifacts/:artifactId` route (`apps/backtester/src/api/server.ts:152`) — **no new endpoint is needed**, contrary to what the 079 design doc speculated. The only missing piece is item 1: getting `evidenceRef` actually populated for lab's real run kind.
3. **The CLI script is a separate, stale legacy path — not the blocker anymore.** `apps/backtester/scripts/produce-evidence.mts:58` still carries `TODO(real-bundle)` and signs a programmatic fixture module (Track B harness), independent of the worker.ts E4 pipeline described above. Worth reconciling/retiring once (1) ships, so there's one signing pipeline, not two.
4. **No consumable SDK export.** `SignedBacktestEvidence`, `canonicalizeEvidenceBody`, and the signing/verify helpers live only in `apps/backtester/src/evidence/{body,canonical,signing}.ts` (app-internal) — zero occurrences of `SignedBacktestEvidence` anywhere under `packages/sdk`. Lab currently vendors its own copy (`src/research/evidence-canonical.ts`, `src/research/evidence-signature.ts`) with a comment flagging the risk: any canonicalization drift between the vendored copy and the real backtester breaks verification silently for real evidence while fixture-based tests keep passing (079 design doc §8 risk).
5. **The pubkey exists — just not where lab or its SDK can get it without hand-copying.** The SPKI PEM for `bt-ed25519-cb1661aa4bcbfff8` is already committed on the **platform** side: `trading-platform/src/admissions/config/trusted-signers.ts:15` (`-----BEGIN PUBLIC KEY-----\nMCowBQYDK2VwAyEA6rqaGPapPlh+k52ZEn3+YDih4PYaqucJMbM3DIbvHdM=\n-----END PUBLIC KEY-----\n`, handshake 2026-06-29). Trading-lab's own repo has zero copies of this PEM. **Actionable today, no backtester code change required:** lab ops can copy that exact string into `LAB_TRUSTED_SIGNERS_JSON='{"bt-ed25519-cb1661aa4bcbfff8":"-----BEGIN PUBLIC KEY-----\n..."}'` right now to unblock local/VPS verification. That's a stopgap, not the fix — two independent hand-copies of a PEM (platform's `trusted-signers.ts`, lab's env) will drift on the next key rotation with no compile-time signal.

## Change

Ask of the **trading-backtester** team (cross-repo item 5 is a **trading-platform**-side pointer, not backtester's to fix):

1. **Add a "sign this real run" mode that doesn't require a curated-baseline twin.** Either (a) generalize the `worker.ts:557-590` E4 block so it fires for ANY completed real run when `deps.evidenceSigningKey` is present and the run's own verdict computation (`computeMetrics` → `decideVerdict`, same as `produce-strategy-evidence.ts:113-121`) yields `'passed'` — decoupled from `curatedBaselineRef`/twin-equivalence, which is a *different* verification concern (strategy-route sandbox parity) that lab's runs don't need; or (b) expose a request-level flag lab's `ResearchPlatformPort.submitStrategyResearchRun` can set (e.g. `requestEvidence: true`) that triggers signing over `buildEvidenceBody({ backtesterRunId, bundleHash: sha256BundleRef(submittedBundleBytes), verdict, datasetRef, window, symbols, timeframe, keyId })` — the exact same body shape `produceStrategyEvidence` already builds, just without the mandatory twin run. Whichever path, the invariant `produce-strategy-evidence.ts` already encodes must hold: **never sign a non-`'passed'` verdict.**
2. **Reconcile `apps/backtester/scripts/produce-evidence.mts`.** Once (1) ships, either delete the CLI script or rewire it to call the same real-run path (closing `TODO(real-bundle)` for real, not adding a second parallel signing implementation).
3. **Export `SignedBacktestEvidence` + `canonicalizeEvidenceBody` from `@trading-backtester/sdk`** (e.g. a new `./evidence` subpath, mirroring how `./builder` and `./intake` are already published) so lab can drop its vendored copy (`src/research/evidence-canonical.ts`) and get canonicalization byte-parity by construction instead of by manual mirroring. Cut one SDK release carrying this.
4. **Publish the pubkey through the SDK, not just platform's source tree** — e.g. export a `TRUSTED_BACKTESTER_SIGNERS: Record<string,string>` (or at minimum the `bt-ed25519-cb1661aa4bcbfff8` PEM as a named constant) from the same `./evidence` subpath, so lab's `LAB_TRUSTED_SIGNERS_JSON` seeds from an SDK constant instead of a hand-copied string that can silently drift from `trading-platform/src/admissions/config/trusted-signers.ts:15` on rotation.

## Acceptance

- A real lab WFO/strategy_baseline holdout run, submitted the way `BacktesterStrategyExperimentRunExecutor` submits it today (no `curatedBaselineRef`), comes back from `GET /v1/runs/:runId/result` with `evidenceRef` populated whenever the run's own verdict is `'passed'`.
- `GET /v1/runs/:runId/artifacts/:artifactId` (existing route) fetches that `evidenceRef` and returns a `SignedBacktestEvidence` JSON body whose `body.bundleHash` equals `sha256` of the bytes lab actually submitted, `body.{datasetRef,window,symbols,timeframe}` equal the submitted run's scope, and `body.keyId === 'bt-ed25519-cb1661aa4bcbfff8'` (or whatever key is currently active).
- `@trading-backtester/sdk` exports `SignedBacktestEvidence` + `canonicalizeEvidenceBody` (and ideally the trusted-signer PEM) from a documented subpath; lab's `src/research/verify-signed-evidence.ts` verifies a REAL signed artifact byte-for-byte using the SDK's canonicalizer (not lab's vendored copy) with zero mismatch.
- A tampered artifact (any byte of `body` changed) still fails verification (`evidence_signature_invalid`) — the invariant this whole slice exists to protect.
