# Design: long_oi as external multi-file code + analyst/critic/researcher eval migration prose → code-golden

- **Date:** 2026-06-30
- **Status:** Approved design (brainstorming complete) → ready for plan
- **Owner:** Alexander Nikolskiy
- **Scope (Slice 1 of 2):** Vendor the platform's `long_oi` strategy module into trading-lab as a **self-contained multi-file `bot_code` fixture** (treated as third-party code, no longer sourced from the sibling platform repo), add a runtime `gatherStrategyCode` helper, and migrate the **analyst + critic + researcher** eval harnesses from the **prose** golden (`GOOD_LONG_OI_PROFILE` / `long-oi-strategy-source.md`) to the existing **code-derived** golden (`src/adapters/builder/fixtures/long-oi-profile.json`). Retire the prose source, the prose golden, and the prose-regen script. **No LLM required** — the slice is fully offline (verified, see §6).
- **Out of scope (Slice 2, separate brainstorm):** the external-code **delivery channel** — how multi-file strategy code physically arrives at lab (gitignored folder drop / archive upload / HTTP ingest endpoint / crawler) and triggers onboard. The `gatherStrategyCode` helper + the "only the profile is persisted" invariant are the seams this slice leaves for it.

---

## 1. Context & problem

The strategy analyst produces a `StrategyProfile` from a source. The **right** input is the strategy **code** — fed the actual `long_oi` module code, the analyst yields a precise, code-grounded reference profile (`long-oi-profile.json`: `sourceKind: bot_code`, confidence 0.99, coreIdea describing the real FSM `IDLE→WATCHING→IN_POSITION→COOLDOWN`, `oiWindow(3)`, `liqAsOf().longUsd`). The lab eval harnesses, however, were built around a **prose** description (`docs/fixtures/strategies/long-oi-strategy-source.md`) and a hand-written prose golden (`GOOD_LONG_OI_PROFILE`, confidence 0.8, generic) — analyzing imprecise prose, not code.

`GOOD_LONG_OI_PROFILE` is consumed by **three** eval harnesses (analyst, critic, researcher). The prose source is also referenced by the analyst-eval `FIXTURES` registry and an old prose-regen script. The code-derived golden already exists (generated by `scripts/regen-from-code.mts` reading `../trading-platform/src/strategies/long_oi`), but the eval harnesses do not use it, and the code lives cross-repo (lab is not self-contained for it).

**Decisions carried from brainstorming:**
- **V1 (self-contained):** vendor the `long_oi` code into lab; `regen-from-code.mts` reads the vendored copy by default; lab eval + regen need no sibling repo.
- **Full retire:** `GOOD_LONG_OI_PROFILE` is removed; all three eval harnesses use the code-golden. Test assertions are re-baselined to the richer code profile.
- Decomposed: this slice is the foundation; the delivery channel is Slice 2.

---

## 2. Goals / non-goals

**Goals**
1. `long_oi` code is vendored into lab as a self-contained, deterministic multi-file `bot_code` fixture; lab no longer reads the platform sibling for it.
2. A reusable runtime `gatherStrategyCode(files, { pathPrefix })` reproduces the exact gather format that produced the code-golden's `sourceFingerprint` (byte-stable), shared by the regen script and (later) the ingestion channel.
3. The analyst eval feeds the **vendored code** (via runtime gather) and scores against the code-golden; the critic and researcher evals use the code-golden as their reference profile. `GOOD_LONG_OI_PROFILE` and the prose source are gone.
4. A guard test pins `sourceFingerprint('bot_code', gather(vendor)) === long-oi-profile.json.sourceFingerprint` (catches drift / a bad re-vendor).
5. Both gates green: `npm run typecheck` exit 0 + `npx vitest run` green.

**Non-goals**
- No external-code delivery channel (Slice 2).
- No change to the `short-pump` fixture (stays prose — out of scope).
- No new LLM call (no golden regeneration — the current platform code already matches the committed golden's fingerprint, §6).
- No change to the runtime two-pass researcher, the onboard handler's contract, or the persistence model (profiles already persist; code is an artifact, not a DB row).

## 3. Design

### 3.1 Vendor the code (self-contained, not compiled)
- Copy `../trading-platform/src/strategies/long_oi/*.ts` (7 files: `flat_phase, manifest, module, params, position_phase, signals, state`) → **`docs/fixtures/strategies/long-oi-code/*.ts`** (byte-identical).
- These are **text fixtures**, NOT compiled: `tsconfig.json` `include` is `["src/**/*","test/**/*","*.config.ts"]`, so `docs/**` is never typechecked; vitest only runs `src|test/**/*.test.ts`. The files are read via `readFileSync`, never imported as modules.
- A short `docs/fixtures/strategies/long-oi-code/README.md` records provenance: vendored from `trading-platform/src/strategies/long_oi`, treated as third-party; to re-vendor, copy the files and re-run the fingerprint guard (and `regen-from-code.mts` only if the code changed).

### 3.2 Runtime gather (`src/domain/strategy-code.ts`)
```ts
export interface StrategyCodeFile { readonly name: string; readonly content: string; }
/** Deterministic multi-file → single bot_code string. MUST reproduce the format that produced
 *  long-oi-profile.json's sourceFingerprint: files sorted by name, each prefixed with a
 *  `// ===== FILE: <pathPrefix>/<name> =====` header, joined by a blank line. */
export function gatherStrategyCode(
  files: readonly StrategyCodeFile[],
  opts: { readonly pathPrefix: string },
): string {
  return [...files]
    .sort((a, b) => a.name.localeCompare(b.name))
    .map((f) => `// ===== FILE: ${opts.pathPrefix}/${f.name} =====\n${f.content}`)
    .join('\n\n');
}
```
- For `long_oi`, `pathPrefix = 'src/strategies/long_oi'`. (Verified: this exact format over the vendored files reproduces `sha256:2bdc…` — §6.)
- `scripts/regen-from-code.mts` is refactored to (a) default `MODULE_DIR` to the vendored dir `docs/fixtures/strategies/long-oi-code` (keeping the `LONGOI_CODE_DIR` env override), and (b) build its `content` via `gatherStrategyCode` (reading the dir into `{name, content}[]`) instead of its private `gatherCode`. Its golden-writing behavior is otherwise unchanged.

### 3.3 Code-golden as the single eval reference (`src/experiments/strategy-analyst/__fixtures__/code-golden.ts`)
```ts
import { readFileSync } from 'node:fs';
import { join, dirname } from 'node:path';
import { fileURLToPath } from 'node:url';
import { AnalystProfileOutputSchema, type AnalystProfileOutput } from '../../../domain/strategy-profile.ts';

const JSON_PATH = join(dirname(fileURLToPath(import.meta.url)), '../../../adapters/builder/fixtures/long-oi-profile.json');
const raw = JSON.parse(readFileSync(JSON_PATH, 'utf8')) as { profile: unknown };
/** The code-derived long_oi analyst profile (sourceKind bot_code). Replaces GOOD_LONG_OI_PROFILE. */
export const CODE_LONG_OI_PROFILE: AnalystProfileOutput = AnalystProfileOutputSchema.parse(raw.profile);
```
- `json.profile` is a full `AnalystProfileOutput` (regen writes `profile: profileOut`); `AnalystProfileOutputSchema.parse` both loads and validates it (drift guard at import time).
- **Replace every `GOOD_LONG_OI_PROFILE` reference** (analyst eval: `scoring/completeness/fabrication/eval-harness/judge.test`; critic eval: `eval-harness/judge/types.test`; researcher `fixtures.ts`) with `CODE_LONG_OI_PROFILE`, and **re-baseline** any assertion that hardcodes a prose-golden value (the code profile is richer/different — counts, specific strings, confidence 0.99 vs 0.8).
- Delete `GOOD_LONG_OI_PROFILE` from `src/experiments/strategy-analyst/__fixtures__/profiles.ts` (and the file if it holds nothing else).

### 3.4 Analyst-eval input = vendored code (`src/experiments/strategy-analyst/fixtures.ts`)
- `FIXTURES['long-oi']` changes from a prose `sourcePath` to the vendored **code directory** + `kind: 'bot_code'`. The harness reads the dir into `{name, content}[]`, calls `gatherStrategyCode(..., { pathPrefix: 'src/strategies/long_oi' })`, and feeds the result as the analyst input; the reference it scores against is `CODE_LONG_OI_PROFILE`. (`resolveFixture` + `FixtureRef` shape extend minimally to carry a code dir; `short-pump` keeps its prose `sourcePath`.)
- The **offline tests stay offline** (no new LLM): the scoring/completeness/fabrication/judge tests exercise the scorer/judge with `CODE_LONG_OI_PROFILE` as the reference (and as a sample profile). The **live analyst-on-code comparison** is the existing paid `--run` path of `strategy-analyst-eval.ts`, not a CI test and not part of this slice's gates.
- `scripts/strategy-analyst-eval.ts` consumes the updated registry; behavior otherwise unchanged.

### 3.5 Researcher-eval fixture (`src/experiments/researcher/fixtures.ts`)
- The profile fixture built from `GOOD_LONG_OI_PROFILE` (fields + `uri: 'docs/fixtures/strategies/long-oi-strategy-source.md'`) is repointed to `CODE_LONG_OI_PROFILE`, with `sourceKind: 'bot_code'` and `uri` pointing at the vendored code dir.

### 3.6 Retire prose
- Delete `docs/fixtures/strategies/long-oi-strategy-source.md`.
- Delete `scripts/regen-long-oi-profile.mts` (the old prose regen; `regen-from-code.mts` is the survivor).
- `long-oi-strategy-research-notes.md` + `long-oi-strategy-rubric.md` are **kept** (input-agnostic scoring/strategy scaffolding); update any line in them that points at the deleted prose source.

### 3.7 Invariant (documented, for Slice 2)
Only the **profile** is persisted to the relational store; the strategy **code** is an artifact (`StrategyProfile.sourceArtifactRef`, kind `strategy_source`), not a DB row. The runtime path is: receive code → `gatherStrategyCode` → store gathered as a `strategy_source` artifact → analyst → persist profile. This slice leaves `gatherStrategyCode` + this invariant as the seam the Slice-2 delivery channel plugs into.

## 4. Affected vs untouched

**Touched:** new `docs/fixtures/strategies/long-oi-code/*.ts` (+ README); new `src/domain/strategy-code.ts` (+ test); new `src/experiments/strategy-analyst/__fixtures__/code-golden.ts`; `scripts/regen-from-code.mts`; `src/experiments/strategy-analyst/{fixtures.ts, __fixtures__/profiles.ts}` + the analyst/critic/researcher eval test files that referenced `GOOD_LONG_OI_PROFILE`; `src/experiments/researcher/fixtures.ts`; a fingerprint guard test. **Deleted:** `long-oi-strategy-source.md`, `scripts/regen-long-oi-profile.mts`, `GOOD_LONG_OI_PROFILE`.

**Untouched:** `short-pump` fixture; `long-oi-profile.json` (the golden — consumed, not regenerated); the math engine / two-pass researcher / onboard handler contract / persistence model; `long-oi-strategy-rubric.md` + `research-notes.md` (kept).

## 5. Testing
- **`gatherStrategyCode`** (unit): sorted, header format, `\n\n` join; deterministic.
- **Fingerprint guard** (test): `sourceFingerprint('bot_code', gatherStrategyCode(readVendor(), { pathPrefix: 'src/strategies/long_oi' })) === long-oi-profile.json.sourceFingerprint`. Fails if the vendor drifts or the gather format changes.
- **`code-golden.ts`** (implicit at import): `AnalystProfileOutputSchema.parse` throws if `json.profile` is malformed.
- **Analyst eval** (existing tests, re-baselined): scores analyst output against `CODE_LONG_OI_PROFILE` over the vendored code; offline/deterministic where it was before (no new LLM).
- **Critic + researcher eval** (existing tests, re-baselined): green against `CODE_LONG_OI_PROFILE`.
- **Both gates:** `npm run typecheck` exit 0 + `npx vitest run` green.

---

## 6. Risks & verification
- **Golden staleness (de-risked):** verified offline that the **current** platform `long_oi` code, gathered with the §3.2 format, yields `sha256:2bdc5389969657cd46ec2500022350e768a0426d8d7bcbb01b14f344157f82b5` — **identical** to the committed golden's `sourceFingerprint` (70863 bytes, 7 files). So vendoring the current code makes the guard pass with **no golden regeneration** (no LLM). If a future re-vendor drifts, the guard fails loudly and a regen (one LLM call) is required.
- **Re-baseline breadth:** `GOOD_LONG_OI_PROFILE` feeds three harnesses; assertions hardcoding prose values must move to the richer code profile. This is mechanical but broad — the plan enumerates each test file; the implementer re-baselines, not reinterprets.
- **`FixtureRef` shape change:** adding a code-dir variant must not break `short-pump` (prose) — keep both source modes.

---

## 7. Success criteria
1. `long_oi` code vendored under `docs/fixtures/strategies/long-oi-code/`, read as text, not compiled; lab self-contained (no platform sibling needed for eval or regen).
2. `gatherStrategyCode` reproduces the golden's fingerprint; the guard test enforces it.
3. Analyst eval analyzes the **code**; analyst/critic/researcher evals reference `CODE_LONG_OI_PROFILE`; `GOOD_LONG_OI_PROFILE` + prose source + prose-regen removed.
4. The "only the profile is persisted" invariant is documented as the Slice-2 seam.
5. typecheck exit 0 + full suite green.
