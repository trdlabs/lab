### 2. Composite `validateStrategyBundle` (chosen: direct import + guard widen)

`src/validation/strategy-bundle-validator.ts`:

```ts
import { validate } from '@trading-platform/sdk/validation';
import { platformContractContext } from '@trading-platform/sdk/research-contract';

export function validateStrategyBundle(a: AssembledStrategyBundle): ValidationVerdict {
  // Half 1 ÔÇö F1 ambient-authority scan (UNCHANGED).
  const violations = AMBIENT_PATTERNS.filter((p) => p.re.test(a.source)).map((p) => p.id);
  if (violations.length > 0) {
    return { status: 'rejected', reason: 'forbidden_ambient_authority', violations };
  }

  // Half 2 ÔÇö SDK-017 manifest-contract gate.
  const { bundleContractVersion, ...manifest017 } = a.manifest; // strip backtester-only field
  const result = validate({ inputKind: 'module', manifest: manifest017 }, platformContractContext());
  if (result.status === 'rejected') {
    return {
      status: 'rejected',
      reason: 'manifest_contract_invalid',
      violations: result.issues.filter((i) => i.severity === 'error').map((i) => i.code),
    };
  }
  return { status: 'valid' };
}
```

- **Composite, both halves:** ambient first (F2-critical untrusted-code gate), then 017 manifest contract. Ambient behavior and its `reason`/`violations` are byte-identical to F1.
- **`reason` for 017 reject:** `'manifest_contract_invalid'` (new stable code, parallel to `'forbidden_ambient_authority'`).
- **`violations`:** the error-severity 017 issue `code`s (e.g. `['lookahead_violation']`), keeping the existing `string[]` shape. `accepted_with_warnings` is **not** a rejection (warnings don't block, per SDK semantics) Ôćĺ falls through to `{status:'valid'}`.
- **`throw` reserved for infra only** (unchanged F1 discipline). `validate` returns a verdict; we never throw on a rejected manifest.
- **Type assignability:** after stripping `bundleContractVersion`, `manifest017` is structurally the platform 017 `ModuleManifest` (both SDKs declare identical field shapes). If `tsc` flags a nominal mismatch at the `validate(...)` call, prefer a narrow typed projection over a blanket `as` cast; a cast is a last resort and must be commented.
