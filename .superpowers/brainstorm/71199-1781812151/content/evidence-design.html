Раздел 2. Evidence Enricher и RAG

Каждый факт имеет источник, свежесть и уровень доверия. RAG не заменяет доменные запросы.

Приоритет источников
1. Exact
Fingerprint, ID, task status
авторитетный факт
2. Structured
Profiles, hypotheses, backtests, bot runs
typed ports
3. Similarity
Embeddings + признаки стратегии
рекомендация, не dedupe
4. RAG
Отчёты, заметки, артефакты
контекст с цитатами

Пакет фактов для стратегии

Subject snapshot
market, timeframe, direction, entry, exit, risk, sourceTextHash
Existing knowledge
exactMatch, similarProfiles[], analysisSummary, hypotheses[]
Observed results
backtests[], botRuns[], activeTasks[], freshness

Как ищем похожие стратегии

Hybrid score = semantic embedding + совпадение рынка/таймфрейма/направления + сходство entry/exit/risk rules

Exact fingerprint решает только идентичность. Hybrid similarity помогает найти смысловые аналоги, но ответ обязан говорить «похожа», а не «та же самая».

Что делает RAG

Разрешено

  • Найти объяснение прошлой гипотезы.
  • Извлечь выводы researcher/critic из артефактов.
  • Подкрепить ответ ссылками на профиль, run и документ.

Запрещено

  • Определять task status из текста.
  • Придумывать метрики при отсутствии run result.
  • Объявлять semantic match точным дубликатом.
Пример ответа до подтверждения
Я понял: long-стратегия отскока на 1m с DCA и двумя тейками.
В базе: точного совпадения нет. Есть 2 похожих профиля; лучший совпадает по входу и направлению, но использует другой стоп.
Предлагаю: создать новый профиль и провести анализ. Это ещё не запускает research или backtest.
