Техника с курса подходит. Важно применить её после точных доменных lookup и к правильным индексам.

Индекс A — Strategy Profiles
Один документ на профиль + нормализованные признаки
  • Поиск похожих торговых идей
  • Metadata: market, timeframe, direction
  • Результат: профиль + similarity evidence
Индекс B — Research Artifacts
Чанки отчётов, гипотез, критики и объяснений
  • Ответы «почему» и «что уже пробовали»
  • Metadata: profileId, taskId, artifact type
  • Результат: фрагменты с source refs

Что не проходит через hybrid search

Exact duplicate
только fingerprint lookup
Metrics/status
только repositories/read ports
Task execution
только confirmation guard

Почему схема сильная

  • BM25 сохраняет точные доменные термины.
  • Vectors находят перефразированные идеи.
  • RRF не требует сравнивать несовместимые scores.
  • Cross-encoder улучшает точность малого финального top.

Ограничения

  • 50/50/20/5 — стартовые параметры, а не закон.
  • Нужны multilingual embedding и reranker для русского/английского.
  • На маленькой базе reranking может не окупить задержку.
  • Качество проверяется eval-набором, не визуально.

Рекомендуемые метрики

Recall@20 после RRF · nDCG@5 после reranker · Precision@5 · false-duplicate rate · p95 latency
