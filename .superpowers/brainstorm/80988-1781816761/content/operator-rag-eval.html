Два eval-контура

CI · каждый PR
  • Vitest fixtures
  • Retrieval metrics без LLM judge
  • Exact duplicate invariant
  • Latency на controlled fixture
  • Regression threshold против baseline
Offline · scheduled/manual
  • Mastra context precision/relevance
  • Faithfulness/answer relevance позже
  • LLM judge на размеченной выборке
  • Human calibration
  • Model/provider comparison

Ragas/DeepEval не входят в v1: Python subsystem добавляется только если Mastra/Vitest не покрывают нужную метрику.

Observability contract сейчас, Phoenix позже

retrievalId · sessionId · subjectHash · source counts · top-K · selected source IDs · degraded reason · stage latency · model/provider · token usage

Сырые стратегии и retrieved chunks не пишутся в audit по умолчанию. Эти поля позже становятся span attributes Phoenix/OpenTelemetry без изменения доменной логики.

Acceptance gates

  • Reranker улучшает nDCG@5 относительно RRF baseline.
  • Recall@20 не ухудшается.
  • p95 end-to-end ≤5s на целевом наборе.
  • Hard deadline 10s никогда не нарушается.

Fail closed / fail soft

  • Semantic result никогда не становится exact duplicate.
  • Judge score не создаёт task.
  • Reranker failure возвращает RRF order.
  • Vector failure возвращает structured + lexical evidence.
